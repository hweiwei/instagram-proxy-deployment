# 第一章：认识你的工具

## 1.1 你需要了解的基本概念

```
┌─────────────────────────────────────────────────────────────┐
│                    互联网访问原理                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   你的设备  ──→  网络请求  ──→  Instagram/TikTok 服务器       │
│      ↑                                      ↓               │
│      │                                 返回数据给你          │
│      │                                      ↓               │
│      └────────────── 代理服务器 ◯ ─────────┘               │
│                                                              │
│                     你在这里           服务器在这里           │
│                     (中国)            (海外服务器)            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 什么是"代理"？

就像你去香港购物需要"代购"一样：
- **代理服务器**: 帮你在海外"买东西"（访问国外网站）
- **IP 地址**: 相当于你的"身份证"，告诉网站你来自哪里

### 为什么要用代理？

```
不用代理：                              用代理：
┌──────────┐                          ┌──────────┐
│  你在中国 │  ──→  Instagram ──→ 拒绝 │  你在香港 │  ──→  Instagram ──→ 欢迎
└──────────┘     (被拒绝)              └──────────┘     (正常访问)
   IP: 北京                              IP: 洛杉矶
```

# 第三章：网络基础入门

## 3.1 理解 IP 地址

```
IP 地址就像你的"网络身份证号"：

┌────────────────────────────────────────────┐
│  IP: 192.168.1.100                        │
│                                            │
│  192.168.1 → 局域网标识（你家路由器）        │
│  100 → 你的设备在这家子的编号              │
└────────────────────────────────────────────┘

公网 IP 示例：
┌────────────────────────────────────────────┐
│  IP: 45.76.123.45                         │
│                                            │
│  45.76.123.45 → 美国洛杉矶 Vultr 服务器    │
│                                            │
│  这个 IP 告诉网站：                        │
│  "你来自美国加州"                          │
└────────────────────────────────────────────┘
```

## 3.2 理解 DNS

```
DNS = 互联网的"电话簿"

当你访问 www.instagram.com：
┌────────────────────────────────────────────┐
│                                            │
│  你输入: www.instagram.com                │
│           ↓                                │
│  DNS 查询: 这个域名对应的 IP是什么？         │
│           ↓                                │
│  DNS 服务器回答: 157.240.27.174            │
│           ↓                                │
│  浏览器访问: 157.240.27.174              │
│                                            │
└────────────────────────────────────────────┘

为什么要改 DNS？
├── 国外 DNS (1.1.1.1) → 解析更快、更准
└── 国内 DNS (114.114.114.114) → 可能被污染
```

## 3.3 理解代理协议

```
常见代理协议（从简单到复杂）：

1. HTTP 代理
   └── 最简单，不能加密，只能代理网页

2. SOCKS5 代理
   └── 比 HTTP 好，能代理更多流量
   └── 不加密，速度快

3. VMess
   └── Xray/V2ray 使用，加密，主流协议
   └── 兼容性好

4. VLESS
   └── Xray 使用，更新更好
   └── 更难被识别，推荐使用

5. Trojan
   └── 伪装成 HTTPS 流量
   └── 防识别能力强

6. WireGuard
   └── 最新技术，速度最快
   └── 适合直播，推荐直播使用
```

## 3.4 理解 TLS 指纹

```
什么是 TLS 指纹？

当你访问 HTTPS 网站时，你的浏览器会发送一个"打招呼"消息：
┌────────────────────────────────────────────┐
│  "你好，我是 Chrome浏览器"                 │
│  "我用 TLS 1.3 协议"                       │
│  "我用 AES-256-GCM 加密"                   │
└────────────────────────────────────────────┘

这个"打招呼"的内容就是 TLS 指纹。

为什么重要？
├── Instagram 会检查这个"打招呼"
├── 如果你的指纹不对，会被怀疑是机器人
└── 所以要用"Chrome 指纹"伪装成正常浏览器
```

# 第五章：客户端配置（图解教程）

## 5.1 Windows 配置 (V2rayN)

### 第一步：下载 V2rayN

```
1. 访问 https://github.com/2dust/v2rayN/releases
2. 找到 "Assets" 区域
3. 下载最新的 zip 文件（文件名含 windows）
   例如：v2rayN-windows-64-With-Core.zip
4. 解压到 C:\Program Files\V2rayN
5. 运行 v2rayN.exe
```

### 第二步：添加服务器

```
1. 打开 V2rayN
2. 点击左上角 "服务器"
3. 选择 "添加 VLESS 服务器"

4. 填写信息（从 /root/proxy-accounts.txt 获取）：

   ┌──────────────────────────────────────┐
   │ 地址 (Address):   45.76.123.45       │
   │ 端口 (Port):     10086              │
   │ UUID:            xxxxxxxx-xxxx      │
   │ 流控 (Flow):     xtls-rprx-vision   │
   │ 传输 (Network):  tcp                │
   │ TLS:             ✓ 开启            │
   │ 指纹 (Security): chrome             │
   │ 别名:            Instagram-Server   │
   └──────────────────────────────────────┘

5. 点击 "确定"
```

### 第三步：连接

```
1. 在服务器列表中，选中刚才添加的服务器
2. 按回车键，或右键选择 "设为活动服务器"
3. 看到右下角出现 "V" 图标，说明连接成功
```

### 第四步：测试

```
打开浏览器，访问：
https://whoer.net

检查显示的 IP 是否为你的服务器 IP
检查时区是否正确
```

# 第七章：养号期攻略（0-30天）

## 7.1 为什么需要养号？

```
TikTok/Instagram 的风控机制：

账号刚注册 = 0分（高风险）
    ↓
慢慢积累信任分
    ↓
30天后 = 基础信任分
    ↓
90天后 = 正常用户

如果你在 0-30 天疯狂操作：
"疯狂关注、疯狂点赞、发帖太多"
    ↓
系统判定：机器人 / 垃圾账号
    ↓
轻则限流，重则封号
```

# 第九章：常见问题与解决方案

## 9.1 连接问题

### Q1: 代理连接成功但无法上网

```
可能原因：
1. DNS 问题
   解决：更换 DNS 为 1.1.1.1

2. 防火墙问题
   解决：在 Vultr 控制台开放端口

3. 账号被封
   解决：检查账号状态
```

### Q2: 代理经常断开

```
可能原因：
1. 服务器不稳定
   解决：换一个服务器节点

2. 网络波动
   解决：使用更稳定的协议 (WireGuard)

3. 同时在线设备太多
   解决：减少连接设备数
```

## 9.2 账号问题

### Q1: 账号被限制功能

```
症状：
├── 无法关注
├── 无法点赞
├── 无法发帖

解决步骤：
1. 立即停止所有操作
2. 等待 24-72 小时
3. 正常浏览，不做任何操作
4. 逐步恢复，先从低频率开始
```

### Q2: 收到验证码要求

```
这是正常的安全验证，不是封号。

解决：
1. 按要求完成验证
2. 验证后降低操作频率
3. 保持正常的代理连接
4. 几天后会恢复正常
```

### Q3: 账号被封禁

```
预防（最重要）：
├── 不要使用任何自动化工具
├── 不要短时间内大量操作
├── 不要发违规内容
├── 保持稳定的 IP

如果被封：
├── 尝试申诉（不一定成功）
├── 分析被封原因
├── 换个账号重新开始
└── 这次要遵守规则
```

## 9.3 TikTok 特殊问题

### Q1: TikTok 无法注册/登录

```
可能原因：
1. IP 被标记
   解决：换一个 IP

2. 设备指纹问题
   解决：恢复出厂设置

3. 手机号问题
   解决：使用 Google Voice 或其他方式
```

### Q2: TikTok 直播卡顿

```
可能原因：
1. 网络延迟太高
   解决：使用 WireGuard

2. 带宽不够
   解决：升级服务器配置

3. 丢包严重
   解决：选择更稳定的节点
```

### Q3: TikTok 直播间没人

```
这是正常现象，新账号直播间没人是常态。

解决方法：
├── 坚持每天直播
├── 内容要吸引人
├── 学会与观众互动
├── 慢慢积累粉丝
└── 配合短视频引流
```
